7. Security Architecture

Security Measures Implemented

  • JWT (HS256) — Stateless auth via httpOnly, sameSite=lax cookies. 7-day expiry.
  • Brute-force protection — Admin accounts locked for 10 min after 5 failed attempts. All attempts logged to security_logs.
  • Role separation — admins table is completely separate from users. Different auth paths.
  • Middleware guard — Every request verified server-side. Role mismatches redirect immediately.
  • File validation — Upload routes check MIME type, file size (max 5MB), and use random filenames.
  • bcrypt (cost 12) — All passwords hashed before storage. Never stored in plaintext.
  • No direct SQL from client — All DB access through parameterised queries via the Turso SDK.
  • Account status checks — pending/suspended/banned users cannot log in with specific error messages.
  • Server Actions — Sensitive mutations (verify user, delete, suspend) use Next.js Server Actions (server-only execution).
